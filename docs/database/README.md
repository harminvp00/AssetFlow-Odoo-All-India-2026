# AssetFlow - Database Configuration & Guide

The primary database is PostgreSQL. AssetFlow uses **Prisma ORM** as the database client and migration manager.

## Local Configuration

The database connection is configured via the `DATABASE_URL` environment variable:

```env
DATABASE_URL="postgresql://<user>:<password>@localhost:5432/assetflow?schema=public"
```

## Schema & Migration Flow

1. **Schema Definition**: The database schema is defined in `server/prisma/schema.prisma`.
2. **Migrations**: Generates SQL migration scripts under `server/prisma/migrations/`.
3. **Prisma Client**: Autogenerated type-safe database queries.

## Standard Development Commands

### 1. Introspect existing database (Reverse Engineer database.sql)

```bash
npx prisma db pull
```

### 2. Generate Prisma Client type declarations

```bash
npm run prisma:generate
```

### 3. Generate and run migrations locally

```bash
npm run prisma:migrate --name init_schema
```

### 4. Seed initial lookup data or defaults

```bash
npm run prisma:seed
```

### 5. Inspect database locally using Prisma Studio

```bash
npx prisma studio
```
